# K-Privacy Filter 발표 용어집

## OPF
OpenAI Privacy Filter의 약자입니다.
텍스트에서 개인정보 후보를 찾는 기본 모델로 사용했습니다.

## Regex Safety Net
정규식 기반 안전망입니다.
주민번호, 전화번호, 주소, 계좌, API key처럼 형태가 비교적 뚜렷한 한국형 개인정보를 보강해서 잡습니다.

## Context Filter
문맥 기반 오탐 제거입니다.
모델명, 예시값, 샘플 데이터처럼 개인정보처럼 보이지만 실제 개인정보가 아닌 경우를 줄이는 단계입니다.

## Span
텍스트 안에서 개인정보라고 판단한 위치 범위입니다.
예를 들어 "최호준"이라는 이름이 10번째 글자부터 12번째 글자까지라면 그 범위가 span입니다.

## Masking
원래 값을 숨기고 `<PRIVATE_PERSON>`이나 `<SECRET>` 같은 placeholder로 바꾸는 작업입니다.

## Precision
시스템이 개인정보라고 잡은 것 중 실제 개인정보인 비율입니다.
높을수록 헛잡기가 적습니다.

## Recall
실제 개인정보 중 시스템이 잡아낸 비율입니다.
높을수록 놓치는 개인정보가 적습니다.

## F1
Precision과 Recall을 함께 보는 균형 점수입니다.
둘 중 하나만 높으면 F1은 크게 오르지 않습니다.

## FPR
False Positive Rate입니다.
개인정보가 아닌데 잘못 개인정보로 잡은 비율입니다.

## Hard-negative
개인정보처럼 생겼지만 실제 개인정보가 아닌 반례입니다.
예: `900101-1234567-A`가 모델명으로 쓰인 경우.

## Adversarial
필터를 속이기 위해 일부러 어렵게 만든 입력입니다.
띄어쓰기, 특수문자, 문맥을 이용해 탐지를 어렵게 만드는 테스트입니다.

## Fine-tuning
기존 모델을 프로젝트 데이터에 맞게 추가 학습하는 작업입니다.
이번 프로젝트에서는 `model.safetensors` 저장까지 성공했지만, 단독 성능이 불안정해 최종 시연에는 쓰지 않았습니다.

## model.safetensors
학습된 모델 가중치를 저장하는 파일 형식입니다.
이 파일이 있다는 것은 학습 결과를 다시 불러올 수 있다는 의미입니다.
